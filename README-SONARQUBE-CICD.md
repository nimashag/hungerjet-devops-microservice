# SonarQube and CI/CD Setup Guide

## Purpose

This guide explains how to:

- Connect this repository to SonarCloud or SonarQube Server.
- Run analysis from the GitHub Actions CI pipeline.
- Upload backend coverage reports from all services, including restaurant-service.
- Use SonarQube for VS Code for local issue detection and connected mode.

## Repository Coverage and Analysis Scope

This project is configured to analyze these source paths:

- delivery-service/src
- orders-service/src
- restaurants-service/src
- users-service/src
- frontend-service/src

Coverage is expected from LCOV files generated by backend Jest runs:

- delivery-service/coverage/lcov.info
- orders-service/coverage/lcov.info
- restaurants-service/coverage/lcov.info
- users-service/coverage/lcov.info

The configuration is defined in sonar-project.properties.

## 1. SonarCloud Project Setup

Use this section if you are using SonarCloud.

1. Create or sign in to SonarCloud.
2. Import the GitHub repository.
3. Confirm these values in SonarCloud project settings:

- Organization key
- Project key
- Default branch

4. Generate a project token in SonarCloud.
5. Save the token in GitHub repository secrets as SONAR_TOKEN.

Recommended repository secrets:

- SONAR_TOKEN
- DOCKER_USERNAME
- DOCKER_PASSWORD

## 2. CI/CD Workflow Behavior

The workflow file is .github/workflows/ci-cd.yml.

Pipeline summary:

1. Build and test backend services in matrix jobs:

- users-service
- restaurants-service
- orders-service
- delivery-service

2. Run frontend build.
3. Run Docker build and push for main/master push events.
4. Run security scan job with SonarQube scan.

Important behavior for Sonar coverage:

- Each backend service runs npm run test:ci --if-present.
- Each backend service uploads coverage/<lcov.info> as an artifact.
- Security scan downloads artifacts and copies them into expected service coverage folders.
- Sonar scanner reads all LCOV files declared in sonar-project.properties.

Restaurant coverage is included when:

- restaurants-service/package.json has test:ci script.
- restaurants-service coverage artifact exists.
- security-scan copy step includes restaurants-service.

## 3. Required Scripts Per Backend Service

Each backend service should define:

- test
- test:ci
- build

Recommended test:ci format:

- jest --coverage --passWithNoTests --ci

Current restaurant-service uses:

- jest --coverage --ci

## 4. GitHub Actions Verification Checklist

After pushing a commit:

1. Open GitHub Actions and select the latest workflow run.
2. Confirm Build users-service, Build restaurants-service, Build orders-service, and Build delivery-service succeeded.
3. Confirm Upload coverage report step exists per backend job.
4. Confirm Security Scan job succeeded.
5. In Security Scan logs, verify coverage files are copied for all backend services.
6. Open SonarCloud project and verify analysis was executed for latest commit.

## 5. SonarQube for VS Code Setup

You can use local analysis and connected mode in VS Code.

### Install the extension

Use one of these methods:

1. VS Code Extensions view:

- Search for SonarQube for IDE.
- Install extension by SonarSource.

2. Command line:

- code --install-extension SonarSource.sonarlint-vscode

### Connect to SonarCloud or SonarQube Server

1. Open Command Palette.
2. Run SonarQube for IDE: Add a new connection.
3. Choose SonarCloud or SonarQube Server.
4. Authenticate with token or browser flow.
5. Bind this workspace to the correct project key.

Connected mode benefits:

- Uses server-side quality profile and rule configuration.
- Synchronizes issue status and suppressions.
- Gives faster feedback before pushing.

## 6. Local Sonar Validation Workflow

Before pushing changes:

1. Run tests with coverage in changed services.
2. Fix critical and blocker issues in VS Code.
3. Confirm no major lint/type errors remain.
4. Push and validate CI.
5. Confirm Sonar dashboard reflects expected issues and coverage.

## 7. Common Problems and Fixes

### Coverage is 0 percent in Sonar

Possible causes:

- test:ci script missing in one or more services.
- LCOV file not generated.
- Artifact not uploaded.
- Coverage copy step omitted one service.
- sonar.javascript.lcov.reportPaths missing path.

Fix:

- Ensure each backend service generates coverage/lcov.info.
- Ensure CI uploads and copies all coverage artifacts.
- Ensure sonar-project.properties includes all LCOV paths.

### Sonar analysis runs but new issues are not visible in VS Code

Fix:

- Confirm connected mode is enabled.
- Confirm workspace is bound to the correct project key.
- Reload VS Code window.

### Quality gate failed even after security fixes

Common causes:

- Coverage threshold not met.
- Duplicated lines threshold not met.

Fix:

- Add targeted tests for new and changed code.
- Refactor duplicated logic.

## 8. Recommended Branch Policy

For stable quality:

1. Require GitHub Actions checks before merge.
2. Require passing Sonar quality gate for pull requests.
3. Block direct push to main when possible.
